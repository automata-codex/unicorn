# Milestone M2.5 — Design Sprint & Base Component Set
## Implementation Spec for Claude Code

**Scope:** Frontend only — no backend work. This milestone establishes the design token layer and base component set that all subsequent frontend work builds on. No feature UI is built here. M3 frontend work must not begin until this milestone is complete.

**Done when:** Token files are in place, all base components are implemented and render correctly against the Mothership theme, and the Storybook-style dev page shows all components and states.

**Reference:** Read `docs/design-system.md` in full before writing any code. It is the authoritative spec for tokens, conventions, and component behavior. This implementation spec covers structure and file locations; `docs/design-system.md` covers visual decisions.

## Part 0: Font Files

IBM Plex Mono is self-hosted. Font files must be in place before the token files are loaded.

**Download source:** https://github.com/IBM/plex/releases — download the latest release archive and extract the `IBM-Plex-Mono/fonts/complete/woff2/` directory.

**Files needed** (copy to `apps/zoltar-fe/static/fonts/`):

```
IBMPlexMono-Regular.woff2
IBMPlexMono-Italic.woff2
IBMPlexMono-Medium.woff2
```

Only these three weights are used. Do not commit unused font files.

The `@font-face` declarations are already in `base.css` and reference `/fonts/` as the path, which resolves correctly from Vite's `static/` directory.

## Part 2: Base Components

All components live at `apps/zoltar-fe/src/lib/components/`. Each is a Svelte 5 component using runes (`$props()`, `$state()` etc.). Each uses only tokens from `docs/design-system.md` — no hardcoded hex values, no primitive token references.

### `Button.svelte`

**Props:**
```typescript
variant?: 'primary' | 'ghost'   // default: 'primary'
disabled?: boolean               // default: false
type?: 'button' | 'submit'      // default: 'button'
fullWidth?: boolean              // default: false
onclick?: () => void
```

**Variants:**

`primary` — uses `--btn-primary-bg`, `--btn-primary-text`, `--btn-primary-radius`. Hover uses `--btn-primary-bg-hover`. Active scales to 0.98.

`ghost` — transparent background, `--btn-ghost-text`. On hover, text becomes `--btn-ghost-text-active`.

`disabled` (applies to both variants) — uses `--btn-disabled-bg`, `--btn-disabled-border`, `--btn-disabled-text`. `cursor: not-allowed`. Does not respond to hover or active states.

**Typography:** `--font-primary`, `--font-size-md`, `--tracking-base`, uppercase.

**Padding:** `--space-4` vertical, `--space-5` horizontal (adjust for `fullWidth`).

---

### `Input.svelte`

**Props:**
```typescript
type?: 'text' | 'number' | 'password'  // default: 'text'
value?: string | number
placeholder?: string
label?: string                          // renders above input if provided
hint?: string                           // renders below input if provided, muted
disabled?: boolean
id?: string
oninput?: (e: Event) => void
onchange?: (e: Event) => void
```

Uses `--input-bg`, `--input-border`, `--input-text`, `--input-placeholder`, `--input-radius`, `--input-padding`.

On focus: `--color-border-input` brightens to `--color-accent-border`. No box shadow. Outline: none.

Label (if provided): `--font-primary`, `--font-size-xs`, `--color-text-tertiary`, `--tracking-wide`, uppercase, `--space-2` margin below.

Hint (if provided): `--font-primary`, `--font-size-xs`, `--color-text-ghost`, `--space-1` margin above.

---

### `Select.svelte`

**Props:**
```typescript
value?: string
options: Array<{ value: string; label: string }>
placeholder?: string
label?: string
disabled?: boolean
onchange?: (e: Event) => void
```

Visually identical to `Input.svelte`. Custom chevron using `--color-text-ghost`. Native `<select>` element with `appearance: none`.

---

### `Card.svelte`

**Props:**
```typescript
padding?: boolean   // default: true — applies --card-padding
```

Uses `--card-bg`, `--card-border`, `--card-radius`, `--card-padding`.

This is a simple wrapper — a `<div>` with card styling and a default slot. No header, no footer logic. Compositors build structure inside it.

```svelte
<!-- Usage -->
<Card>
  <SectionLabel>IDENTITY</SectionLabel>
  <!-- content -->
</Card>
```

---

### `SectionLabel.svelte`

A single-slot component for section headers (IDENTITY, STATS, SAVES, etc.).

Uses `--label-text`, `--label-size`, `--label-tracking`. Renders as a `<div>`. No props — slot content only.

```svelte
<!-- Usage -->
<SectionLabel>SURVIVORS</SectionLabel>
```

---

### `ResourceBar.svelte`

The HP and Stress bar used in the play view status panel.

**Props:**
```typescript
label: string           // e.g. 'HP', 'STR'
current: number
max: number
color?: 'hp' | 'stress' // default: 'hp'
```

Track uses `--color-bar-track`. Fill uses `--color-hp` or `--color-stress` per `color` prop.

Width of fill: `(current / max) * 100%`. Clamp between 0% and 100%.

Height: 3px. Border radius: `--radius-full`.

Label: `--font-primary`, `--font-size-xs`, `--color-text-ghost`, `--tracking-wide`, uppercase.

Value text (`current/max`): `--font-primary`, `--font-size-xs`, color matches fill color.

---

### `StatusDot.svelte`

Presence indicator — the small colored dot in the play view status strip.

**Props:**
```typescript
status?: 'online' | 'offline'   // default: 'online'
```

6×6px circle. `online`: `--color-presence-online`. `offline`: `--color-text-ghost`.

## Constraints

- No hardcoded hex values anywhere. All colors via semantic tokens.
- No primitive token references in components. Semantic tokens only.
- No Tailwind or utility class framework.
- All components are Svelte 5 with runes.
- Mobile-first: components must render correctly at 375px viewport width.
- Bits UI is available for accessibility primitives but none of the above components require it — these are all simple enough to implement directly.
